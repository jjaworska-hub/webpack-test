Formularze

Znacznik form jest swego rodzaju opakowaniem dla naszego formularza. Podstawowym elementem, który buduje nasz formularz jest input. Każdy input powinien mieć swoją nazwę (name). Należy dodać też id (może mieć ono taką samą nazwę jak name). Nie stylujemy za pomocą id! ID jest potrzebne do indentyfikacji a nie do stylizowania.
Znacznik label tworzy nam znacznik naszego pola. Dodając do label wyrażenie for="id_inputu" sprawiamy,że label jest przyczepiony do inputu (klikając na niego aktywujemu pole input).
Domyślnym typem inputa jest tekst. Kolejnym atrybutem inputa jest value (wartość). Uzupełni on nasze pole o wskazaną wartość. Istnieje również atrybut o nazwie placeholder, która służy do wyświetlenia podpowiedzi w okienku inputa np. "Please enter your name".

Kolejnym znacznikiem jest fieldset, który grupuje pola inputu. Innym typem naszego inputa jest typ radio (type="radio"), które jest takimi "guzikami" w formularzu. Co jest ważne w radio buttonach to to,że name dla całej grupy musi być taki sam.

Następny typ to checkbox (type="checkbox"). Jest to taki element,który możemy "zaptaszkować". Ciekawostka: jeśli chcemy aby w naszym formularzy niektóre radio i checkboxy były zaznaczone możemy użyć atrybutu checked. Nie trzeba dopisywać do niego dodatkowych name, id itp..

Typami, które pomagają nam spredzyzować jakie dane chcemy aby zostały wpisane w naszym formularzu są number,tel,email.

Kolejny typ to typ range. Zapisujemy go w następujący sposób:

input type="range" min="0" max="100" step="5"

Typ password używamy do haseł.

Typ file służy do przekazywania plików. Np. awatary.

Kolejny element to element select. Select pomaga nam w zrobioniu takiej "listy do wyboru". Zawiera w sobie taki atrybut jak option.

Następnym znacznikiem jest textarea, który tworzy pole do wpisywania tekstu. Np. "O mnie.." itp..

Typ submit (input type="submit") jest typem inputa, który będzie wysyłać nam nasz formularz. Submit jako jedyny nie zawiera labela.

Aby wysłać gdzieś nasz formularz potrzebujemy atrybutu a nazwie action (action="url://z-którym-gadamy"). Do sprawdzenia czy formularz działa poprawnie możemy użyć strony Form Checker.

Domyślną metodą wysyłania formularzy jest metoda get (method="get"). Oprócz niej istnieje jeszcze metoda post (method="post"). Post działa w ten sposób, że adres URL "jest czysty". Jest to bardziej bezpieczniejsze, ponieważ nie da się zmodyfikować treści lub ominąć formularz.

Istnieje też atrybut jak enctype, które wartością domyślną jest "application/x-www-form-urlencoded". Jednak jeśli chcemy wysyłać pliki powinniśmy zmienić tę wartość na "multipart/form-data".

Walidacja formularzy

Walidacja to sprawdzanie poprawności danych wprowadzanych do formularzy.
Zacznijmy od atrybutu zwanego required. Atrybut sprawia,że formularz nie zostanie wysłany dopóki użytkownik nie uzupełn wymaganego pola/pól. Dopisujemy required do pól formularza, które mają być konieczne do wysłania formularza.

Inną formą walidacji są tzw. patterny z wykorzystaniem wyrażeń regularnych. Wyrażenia regularne jest to pewien rodzaj zapisy pozwalający na wykrywanie pewnych ciągów znaków.

Tworzenie przycisków

W języku HTML przycisk zapisujemy przy pomocy atrybutu button. Róznica między button a a jest taka,że button odpowiada za interakcje na stronie, natomiast "a" odpowiada za nawigację (skakanie po sekcjach, stronach, wchodzenie na strony zewnętrzne)

Warto zapamiętać

  • Jeśli coś jest powtarzalne na podstronach wstawiamy to przed "main"

  • Nie używamy tabel do budowania layoutów!

  • Atrybut placeholder nie zastępuje elementu label!

  • Walidacja po stronie frontendu nie zastępuje walidacji po stronie backendu!
